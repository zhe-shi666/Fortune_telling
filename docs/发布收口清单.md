# 发布收口清单

用于后续每次接近上架时逐项核对，避免只关注代码完成度而忽略审核、资源与文案边界。

## 1. 产品与文案

- [ ] 所有命理结果都统一表述为娱乐参考或辅助参考
- [ ] 不存在“保证”“精准预测”“专家背书”等高风险承诺文案
- [ ] 不存在开发口吻、测试口吻、演示口吻或技术化错误提示
- [ ] 所有按钮都有明确结果，不存在死按钮或空跳转

## 2. 充值与商业化

- [ ] 决定本版是否开放真实内购
- [ ] 若未开放，充值入口文案与审核说明一致为“仅展示购买说明，不触发真实支付”
- [ ] 若开放，完成 StoreKit、商品配置、购买恢复与失败处理
- [ ] 不存在把未来能力伪装成已可购买的描述

## 3. UI 与体验

- [ ] 6 个 MVP 页面均完成空态、加载态、错误态检查
- [ ] 小屏尺寸不溢出
- [ ] 大字号模式无严重重叠
- [ ] 底部导航、弹层、按钮反馈风格统一
- [ ] 长文案、签语、结果卡片滚动体验正常

## 4. 数据与隐私

- [ ] 命主档案、姓氏、收藏等本地存储边界清晰
- [ ] 已准备正式隐私政策 URL
- [ ] 已填写 App Store Connect 隐私信息
- [ ] 若未来接入后端或内购，补齐新的数据流转说明

## 5. 工程与资源

- [ ] `Assets.xcassets` 已存在并包含正式 `AppIcon`
- [ ] `Bundle Identifier`、签名团队、版本号已切换到正式配置
- [ ] 若不支持 iPad，已在工程配置中明确关闭
- [ ] 真机安装、冷启动、热启动验证通过

## 6. 提审前检查

- [ ] 准备 App 名称、副标题、描述、关键词、截图
- [ ] 准备支持链接与隐私政策链接
- [ ] 准备审核备注，说明 App 核心用途与未开放能力
- [ ] 完成至少一轮 TestFlight 验收
